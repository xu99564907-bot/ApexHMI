## 1. 项目概览

**ApexHMI** 是一个基于 **.NET Framework 4.8 + WPF** 的工业 HMI（人机界面）应用，主要面向 PLC 设备的 OPC UA 通信、IO 表管理、报警监控、可视化设计等场景。

### 1.1 技术栈

| 组件 | 技术 | 版本 |
|---|---|---|
| 框架 | .NET Framework 4.8 + WPF | - |
| MVVM | CommunityToolkit.Mvvm | 8.4.2 |
| DI 容器 | Microsoft.Extensions.DependencyInjection | 10.0.5 |
| OPC 通信 | OPCFoundation.NetStandard.Opc.Ua | 1.5.378 |
| UI 主题 | MaterialDesignThemes | 4.9.0 |
| CSV | CsvHelper | 33.1 |
| JSON | System.Text.Json | 10.0.5 |
| 日志框架 | ❌ 无 | - |
| 测试框架 | ❌ 无 | - |

### 1.2 规模

- C# 文件：105
- 代码行数：约 18,690

## 4. 优化路线图

### 阶段 1 · 止血（Week 1）
让线上问题可追踪、不丢数据、不崩溃。
- 引入 Serilog（文件 + 滚动归档）
- 全局异常处理（Dispatcher / AppDomain / TaskScheduler）
- 消灭 `async void`（事件处理器统一模板）
- 凭据加密（DPAPI）

### 阶段 2 · 解耦 MainViewModel（Week 2-4）
- 按模块拆分为独立 ViewModel
- MainViewModel 退化为外壳 (< 500 行)
- 用 IMessenger 替代直接引用

### 阶段 3 · 服务层重构（Week 4-6）
- 全 Service 抽接口
- 拆分 OpcUaService → ConnectionManager / NodeResolver / SubscriptionManager / ReadWriteService
- 拆分 IoTableImportService → 按格式 strategy
- 统一线程同步策略

### 阶段 4 · 测试体系（Month 2）
- 新建 ApexHMI.Tests（xUnit + Moq + FluentAssertions）
- Service 层目标 60% 覆盖
- VM 关键命令覆盖
- 接 CI

### 阶段 5 · UI / 体验（Month 2-3）
- code-behind 业务搬到 VM
- 抽自定义控件 / DataTemplate
- Converter 颜色集中到 ResourceDictionary
- 大集合虚拟化、修内存泄漏

### 阶段 6 · 配置与工程化（持续）
- 合并 config 为单 appsettings.json + schema
- IOptions<T> 注入
- 魔法数迁移配置
- i18n、Roslynator/StyleCop
