# ApexHMI 线程模型规范

> 适用范围：本仓库所有 Service / ViewModel / View
> 目的：在没有正式并发测试的情况下，用统一约定避免死锁、竞态、UI 卡顿
> 现状：项目同时使用 `lock` / `SemaphoreSlim` / `ConcurrentDictionary` / 无锁，约定不一致

---

## 1. 三类执行上下文

| 上下文 | 用途 | 调度器 | 常见入口 |
|---|---|---|---|
| **UI 线程** | 修改 ObservableCollection / 设置 ObservableProperty / 弹窗 | `Application.Current.Dispatcher` | XAML 事件、Command 默认 continuation |
| **OPC 工作线程** | OPC UA 长 IO（连接、订阅、批量读写） | `Task.Run` 或 OPC SDK 内部线程池 | `OpcUaService.*Async` |
| **磁盘 IO 工作线程** | JSON / CSV / XLSX 读写 | `Task.Run` 或 `await File.*Async` | `*Service.SaveAsync` / `LoadAsync` |

UI 线程**绝不允许**执行下面任一：
- 同步 OPC 调用（`*Async().Result`、`.Wait()`）
- 大文件解析 / 模板生成
- `Thread.Sleep`

工作线程**绝不允许**直接：
- `Add` / `Remove` ObservableCollection（必须 marshal 回 UI）
- 写 ObservableProperty（除非该属性所在 collection 已通过 `BindingOperations.EnableCollectionSynchronization` 开启同步）

## 2. 同步原语选型

| 场景 | 选用 | 理由 |
|---|---|---|
| **跨 await 的临界区** | `SemaphoreSlim(1, 1)` + `await WaitAsync()` | `lock` 不能跨 await |
| **纯同步、短临界区**（< 100 行字节码、不调 IO） | `lock (privateObj)` | 性能最好、零分配 |
| **UI 集合并发读写** | `BindingOperations.EnableCollectionSynchronization(coll, lockObj)` | WPF 绑定层会代你拿锁 |
| **多读少写、键值缓存** | `ConcurrentDictionary<TKey, TValue>` | 内置无锁 / 分段锁 |
| **跨方法状态机 / 取消** | `CancellationTokenSource` + `CancellationToken` 参数 | 标准模式 |

> ⚠️ **不要混用**：同一份共享数据只用一种原语保护。`OpcUaService` 当前混用了 SemaphoreSlim（`_cacheIoLock`）、object lock、ConcurrentDictionary，应在 P1-OPC 拆分时统一为「`SemaphoreSlim` + `ConcurrentDictionary`」组合。

## 3. async / await 约定

1. **公开异步 API 一律 `Task` / `Task<T>`**，不允许 `async void`，事件处理器除外（必须用 [AsyncEventHandler.RunSafe](../Views/Common/AsyncEventHandler.cs) 包装）。
2. 库 / Service 层 `await` 默认加 `.ConfigureAwait(false)`（避免不必要 marshal 回 UI 线程）。
   ViewModel 顶层 `await` **不要** 加 `ConfigureAwait(false)`，因为命令通常需要回到 UI 线程更新属性。
3. 长 CPU 任务（解析、模板生成）必须 `await Task.Run(() => ...)`。
4. 取消令牌：所有 IO 类公开 API 提供 `CancellationToken cancellationToken = default` 参数。
5. 绝不在锁内 `await`（包括 `lock`、`Monitor.Enter`）。

---

## 4. ObservableCollection / ObservableProperty 写入规则

### UI 线程写入（默认）
```csharp
// MainViewModel
Tags.Add(newTag);          // OK，命令在 UI 线程上下文
SystemMessage = "...";     // OK
```

### 工作线程写入（必须 marshal）
```csharp
// 在 Service 回调或 Task.Run 内部
await RunOnUiThreadAsync(() => Tags.Add(newTag));
```

### 大批量更新（推荐）
1. 在工作线程构建 `List<T>`
2. UI 线程一次性 `Clear()` + `AddRange`（CommunityToolkit 提供）
   或：在 ViewModel ctor 调一次 `BindingOperations.EnableCollectionSynchronization(Tags, _tagsLock)`，之后任意线程可直接修改。

---

## 5. 计时器约定

- WPF 内 `DispatcherTimer`：tick 在 UI 线程，适合刷新状态、轮询 UI 数据
- `System.Threading.Timer`：tick 在线程池，适合后台周期任务
- **每个 Timer 必须配对 `Stop()` + `Dispose()`**（在窗口 Closed / VM 销毁时）
