## 一、程序生成 — 总览

软件主导航「**设计器**」分组下有三个子页：

| 子页 | 用途 | 输出 |
|---|---|---|
| 手动程序生成 | 基于 IO 表，按命名规则一键生成 PLC `*.st` 源代码 | DB IO 块 / DI-DO Mirror / ACT_Comment |
| 自动程序生成 | 基于 SFC 步骤集合，生成自动流程 PLC 代码 | OPXX_Graph 自动主程序 |
| 初始化程序生成 | 与自动程序对称，独立步骤集合，生成初始化流程 | OPXX_Graph 初始化主程序 |

> 三个子页右上角带有「**手动程序生成 / 自动程序生成 / 初始化程序生成**」横向切换按钮，互相切换不丢未保存数据。

---

### 1.1 手动程序生成

#### 操作流程

1. **PLC 模板**：左栏下拉选择目标 PLC 型号（默认"汇川中型 PLC"）
2. **工位号 / OP 编号**：填工位号，如 `OP10`、`OP70`
3. **保存间隔（分钟）**：小于 5 分钟时，重复保存会写入同一文件并追加履历
4. **导入 IO 表**：点击「导入 IO 表」按钮选 Excel / CSV 文件
   - 自动解析输入地址 / 输入注释 / 输出地址 / 输出注释 4 列
   - 解析后在「IO 表预览」DataGrid 中显示
5. **校验与生成**：点「**生成程序**」按钮
   - **D1 自动校验**：地址重复 / 类型不匹配的行会被高亮，生成被阻止
   - 校验通过后生成 PLC 代码，落到 `<PLC 程序保存目录>/2.Main/Main_IO.ST` 等约定路径
6. **预览结果**：右栏「程序预览」面板：
   - 文件下拉选择刚生成的具体 `.st` 文件
   - 「**对比 Git**」（D2）：弹出 side-by-side 对比窗，红/绿色标差异
   - 「**复制当前文件**」：复制内容到剪贴板
   - 「**打开当前文件**」：用系统默认编辑器打开

#### 多工位批量生成（D3）

左栏底部「**多工位批量生成**」卡片：
- 输入 CSV 工位号清单，如 `OP30,OP40,OP50`
- 点「批量生成」逐个切换 OP 后跑生成
- 适合一次性给整条产线所有工位生成代码

---

### 1.2 自动程序生成（SFC 步骤式）

按"步骤 → 动作 → 跳转"组织的 SFC 流程，生成 OPXX_Graph 自动主程序。

#### 操作流程

1. **左栏选择工位号**（与手动程序的 OP 编号联动）
2. **添加步骤**：中栏「SFC 步骤列表」DataGrid
   - 「+ 添加步骤」按钮新增一步
   - 每步可填：StepNo / 动作清单 / 跳转条件
3. **编辑动作（每步的右侧 ActionSummary）**：
   - 双击 ActionSummary 单元格 → 弹「动作编辑」对话框
   - 支持的动作类型：气缸、轴、机械手、挡停、电机、报警、延时、条件分支
   - 每条动作配 DeviceName + ActionType + 可选参数
4. **编辑跳转（NextSummary）**：
   - 默认顺序跳到下一步
   - 可改为条件跳转：`Step5` 或 `IF DB1050.X.Done THEN Step3 ELSE Step8`
5. **生成代码**：右栏「**SFC 程序预览**」按钮
   - 自动按 SFC 模板渲染 ST 代码
   - 落到 `OPXX_Graph_ACT_AutoRunST<N>.st` 文件
6. **代码差异对比**：右栏顶部行点击拖动 → 显示「**对比 Git**」按钮

#### 报警术语本地化

中栏顶部「**编辑中文→英文报警术语对照表**」按钮：
- 弹「报警术语词典」对话框
- 中文报警 → 英文别名映射
- 生成代码时自动按映射翻译注释

---

### 1.3 初始化程序生成

与「自动程序生成」结构对称，独立步骤集合，生成 **OPXX_Graph 初始化主程序**。

操作与 §1.2 完全相同，唯一区别：
- 输出文件名是 `OPXX_Graph_ACT_InitST<N>.st`（非 AutoRunST）
- 步骤集合独立维护，与自动程序互不影响

> 初始化流程常用于：上电自检、回原点、参数加载、连接外设。

---

### 1.4 Git 代码同步

完整的 Git 管理面板位于「**手动程序生成**」左栏的「**Git 代码拉取**」Expander，提供从远程 Git 仓库**克隆 / 拉取 / 自动追加生成代码 / commit / push** 的全套流程。详见 §五。

#### 简要操作（5 步快速上手）

1. 展开「Git 代码拉取」Expander
2. 填**仓库地址**（如 `https://github.com/your/repo.git`）+ **保存目录**（本地路径）
3. 私有仓库展开「**认证**」填用户名 + Personal Access Token
4. 点「**拉取代码**」按钮 → 首次会 clone、后续 pull
5. 之后每次「生成程序」自动按映射规则追加到本地 Git 目录的对应 `.st` 文件

详细字段、自动 commit、push、冲突处理、映射规则、错误诊断等见 [第五章：Git 代码管理（完整版）](#五git-代码管理完整版)。

## 二、画布设计 — 总览

主导航「**设计器 → 画布设计**」子页。这是构建 HMI 运行时画面的核心工作区。

### 2.1 工作区结构

```
┌─────────────────────────────────────────────────────────┐
│  顶部：[工具箱] (左) │ [动作按钮组] (右)                  │
├──────────┬─────────────────────────────────┬────────────┤
│ 左栏      │                                 │  右栏       │
│ 工程信息   │      中央画布                    │ 属性编辑器   │
│ 页面树    │  (拖拽控件、缩放、网格、对齐)      │ (4 Tab)     │
│ 库 (Lib)  │                                 │            │
│ 资源      │                                 │            │
└──────────┴─────────────────────────────────┴────────────┘
```

#### 顶部按钮组

| 按钮 | 功能 |
|---|---|
| 🗄 **PLC 变量** | 从 PLC 程序保存目录加载 `Device.Application.xml`，按 OP 工位号筛选导入变量表 |
| 📁 文件夹 | 手动选择 PLC 变量 XML 文件 |
| 🎨 **样式** | 全局样式编辑器（色板 + 字体预设） |
| 🔖 收藏 | 把当前选中控件存到项目库 |
| 🌍 全局库 | 把当前选中控件存到全局库（跨工程共享） |
| 🌐 **文本** | 多语言文本资源编辑 |
| 🌐 zh-CN | 当前语言切换下拉 |
| 📋 **列表** | 文本列表 / 图形列表资源编辑 |
| 🧩 **Faceplate** | 切换到 Faceplate 编辑模式 |
| 📦 **导出** | 工程打包为 zip |
| 📥 **导入** | 从 zip 导入工程 |
| 〰 **模拟** | 离线模拟开关（注入 SimTag 假数据） |
| 🚀 **发布** | 保存工程并推送到运行时 |

### 2.2 控件工具箱

工具箱按 WinCC 风格分 5 大组：

| 组 | 包含控件 |
|---|---|
| **基本对象** | 文本 / 矩形 / 椭圆 / 直线 / 折线 / 多边形 / 图形视图 / 数字 I/O / 符号 I/O / 图形 I/O / 日期时间 |
| **元素** | 按钮 / 开关 / 圆形按钮 / 棒图 / 量规 / 滑块 / 滚动条 / 时钟 / 组合框 / 列表框 / 复选框 / 单选 |
| **控件** | 趋势视图 / 报警视图 / 表格视图 / 画面窗口 |
| **媒体/分析** | HTML 浏览器 / PDF 视图 / 媒体播放器 / XY 趋势 / 报表视图 |
| **业务** | 配方视图 / 用户视图 / 系统诊断 / 报警指示器 / 状态强制 |
| **我的 Faceplate** | 内置气缸 / 内置轴 / 内置机械手 / 内置挡停 + 用户自定义 |

#### 工具栏 📌 悬浮 / 固定

工具箱右上角有「📌 PinOutline / Pin」图钉按钮：
- **固定模式**（默认）：工具箱在顶部 Row 0
- **悬浮模式**：工具箱浮起来可拖动（Thumb 拖把柄），允许移到画布上方任何位置

---

### 2.3 拖拽与定位

#### 基础操作

- **拖控件到画布**：从工具箱按住左键拖到画布释放，自动添加到鼠标释放点
- **拖变量到画布**：从左栏「PLC 变量」树拖变量到画布
  - 弹候选菜单（按变量类型）：
    - BOOL → 按钮 / 指示灯 / 开关 / 复选框
    - INT/REAL → I/O 域 / 棒图 / 量规 / 滑块 / 数值显示
    - STRING → 文本域 / 符号 I/O 域
  - 选定后自动创建控件 + 完成变量绑定
- **选中**：左键点击控件
- **多选**：Ctrl + 点击、或鼠标空白处拖框选
- **移动**：选中后拖动
- **调整大小**：选中后边缘出现 8 个手柄，拖手柄

#### 画布工具

| 工具 | 位置 | 功能 |
|---|---|---|
| **网格** | 画布顶部 | ☑ 显示网格（间距可调） |
| **吸附** | 同上 | ☑ 自动吸附到网格 |
| **缩放** | 同上下拉 | 50% / 75% / 100% / 125% / 150% / 200% |
| **适应窗口** | 缩放右侧 | 自动按画布尺寸缩放；再次点击恢复 100% |
| 鼠标坐标 | 同上 | 实时显示当前鼠标在画布坐标 |
| 控件数 | 同上 | 当前画布上 widget 总数 |

#### 多选对齐

选中 2+ 控件后画布顶部出现对齐工具栏：
- **左对齐 / 右对齐 / 顶对齐 / 底对齐**
- **横向均布 / 纵向均布**：3+ 控件时按等间距
- **等宽 / 等高**：以选中第一个为基准
- **删除**：批量删除选中

#### 双击画布弹出独立窗口

双击画布空白处 → 整个设计器（含工具栏 + 页面树 + 画布 + 属性编辑器）弹到独立 Window，关闭弹窗自动恢复。便于多屏操作。

### 2.4 属性编辑器

右栏顶部 **4 Tab**：

| Tab | 内容 |
|---|---|
| **属性** | 当前选中控件的所有 PropertyDescriptor，按 Category 分组（常规/外观/格式/限值/安全/闪烁/字体/边框/布局/...） |
| **动画** | 6 类动画配置（外观/可见性/4 种移动） |
| **事件** | 6 个生命周期事件 × 多动作链 |
| **文本** | 当前控件用到的多语言文本 |

#### 字段类型化编辑器（属性 Tab）

不同字段类型自动渲染对应编辑器：

| 字段类型 | UI |
|---|---|
| String / Number / Integer | 普通 TextBox |
| Boolean | True/False ComboBox |
| Color | 色块预览 + hex 输入 + Popup（8 工业色 / 6 样式引用 / 自定义） |
| TagAddress | TextBox + 🔍 + **TagAutoComplete**（输入时弹下一级候选） |
| PageRoute | ComboBox（固定页 + 用户页） |
| Enum | ComboBox（选项带中英对照） |

---

### 2.5 数据绑定

「属性」Tab 顶部「**数据绑定**」分组：
- **读取地址（订阅）**：控件订阅此变量，PLC 端值变化时控件自动刷新
- 留空 = 不订阅，控件只响应点击不显示状态

读取地址 ≠ 写入地址。写入地址在「事件」Tab 的动作步骤里独立配置（双地址模型）。

#### Quality Code 显示

订阅的 OPC UA 变量带质量信号：
- Good → 正常显示
- Uncertain → 值末尾加 `⚠`
- Bad → 显示 `####` + 灰底 + 红边框（数据不可信）

---

### 2.6 事件与动作

「事件」Tab 左侧 6 个事件触发点：

| 事件 | 触发时机 |
|---|---|
| 单击 | 鼠标按下→释放都在控件内 |
| 按下 | 鼠标按下瞬间 |
| 释放 | 鼠标松开瞬间 |
| 激活 | 控件获得焦点（Tab 流转到它） |
| 取消激活 | 失去焦点 |
| 数值更改 | 输入控件值变化时 |

#### 每个事件挂动作链

右侧动作列表卡片：
- **↑↓** 排序
- **×** 删除
- **＋ 新建动作**：弹「系统函数选择」分类菜单：
  - **编辑位**：置位 / 复位 / 取反位 / 按下时置位
  - **写入变量**：写入布尔 / 整数 / 浮点 / 加 1 / 减 1
  - **画面**：跳转页面（固定页/用户页）/ 返回 / 打开弹窗
  - **报警**：确认当前报警 / 清空报警缓冲
  - **其它**：显示对话框 / 播放声音

每个动作步骤按其类型自动生成参数编辑器：
- TagAddress 字段自动挂 TagAutoComplete
- PageRoute 字段走 NavigateTargets 下拉
- Bool / Number 字段对应类型化编辑器

#### 按钮特殊模式

按钮（button）控件支持 **Push / Toggle 双模式**：
- **Push**：点击发一次性事件
- **Toggle**：点击读 stateTag 后写反值，View 按 stateTag 当前值显示 onText / offText / onPicture / offPicture

#### 写 PLC 确认 + 审计

所有写入操作走「**WriteWithConfirm**」流程：
- 3 秒超时（可改）
- 失败 → 顶部系统消息显示"写入失败：{tagId} ({错误码})"
- 成功失败都写审计 trail（SQLite `data/audit.db`，90 天自动滚动）

### 2.7 动画系统

「动画」Tab 左侧 8 项导航：

| 类别 | 说明 |
|---|---|
| 总览 | 列出已配置的动画 + 跳转 / 删除 |
| **外观** | 变量 + 类型（范围 / 多位 / 单位）+ 行表（范围/背景/前景/闪烁） |
| **可见性** | 变量 + 条件（True / False / 范围内）+ 否则隐藏 / 禁用 |
| **水平移动** | 变量值映射到 X 像素范围 |
| **垂直移动** | 同上 Y |
| **直接移动** | XTag + YTag 双变量直接驱动 |
| **对角线移动** | 同时双变量映射 |

#### 外观动画（范围式）举例

变量 `DB1050.Cmd.TeachOn`，类型范围：

| 范围 | 背景色 | 前景色 | 闪烁 |
|---|---|---|---|
| 0 | 灰 | 黑 | 否 |
| 1 | 绿 | 黑 | 否 |
| 2 | 黄 | 黑 | 是 |
| ≥3 | 红 | 白 | 是 |

值落到 2 → 控件背景变黄并闪烁；值变 0 → 恢复灰底。

闪烁全局 300ms 时钟同步（多控件闪烁不会节奏不同步）。

#### 字段动态化 🔶

属性 Tab 里每个颜色/文本字段右侧的 🔶 小图标：
- 灰色 = 静态值
- 橙色 = 已绑变量
- 点击切换 → 弹卡片选「静态 / 绑变量 / 绑脚本」

### 2.8 多语言文本

「文本」Tab 列出当前控件用到的所有 `{text:xxx}` 引用。

#### 引用语法

控件 Text / Tooltip 字段填 `{text:welcome}` → 运行时按当前语言查 `Document.TextResources.Entries["welcome"][currentLang]`。

#### 编辑入口

顶部「**文本**」按钮 → 「**多语言文本资源**」对话框：
- 左侧 key 列表
- 右侧多语言列（zh-CN / en-US / ja-JP …）
- 增 / 删 / 改 key
- 顶部「**zh-CN**」下拉切换当前显示语言，所有控件 `{text:xxx}` 引用即时刷新

### 2.9 Faceplate（自定义复合控件）

WinCC 风格的复合控件封装。

#### 创建 Faceplate

1. 顶部「**Faceplate**」按钮切换到 Faceplate 编辑模式
2. 左栏从「页面列表」切换为「Faceplate 列表」，点 **+ 新建 Faceplate**
3. 填写名称 / 分类 / 默认尺寸 / 图标
4. 中间画布编辑 Faceplate 的**内部画面**（可拖普通 widget 进来）
5. 右栏「**接口属性**」分组添加接口参数：
   - Key（如 `deviceName`）
   - 类型（String / Number / Boolean / TagAddress / Color / PageRoute）
   - 默认值
6. 内部 widget 的字段用 `{prop:deviceName}` 引用接口属性

#### 实例化

退出 Faceplate 编辑模式 → 工具箱出现「**我的 Faceplate**」分组 → 拖入画布生成实例。

实例属性 Tab 显示「接口属性」分组，按 type 自动渲染对应编辑器，填实参值。

#### 版本管理

## 四、常见问题

### Q1：导入 PLC 变量很慢 / 卡死

A：第一次导入按 OP 工位号筛选——只勾选你工程实际用的 OP，避免一次性加载全部数十万变量。
- 顶部「PLC 变量」按钮 → 弹工位选择对话框 → 默认全选改为只选用到的几个 OP
- 设置保存到 `config/plc-variable-import.json`，下次启动自动按上次选择加载

### Q2：写 PLC 不生效

A：检查：
1. **质量信号**：控件是否变 `####`？变了说明 PLC 连接异常
2. **顶部系统消息**：是否提示"写入失败：tagId (错误码)"？
3. **审计日志**：打开 `data/audit.db`，查 success=0 的最近记录拿错误详情
4. **权限**：控件属性是否设了 `authorization`？当前用户角色不匹配会拒

### Q3：旧工程加载后控件显示 `[typeId]` 占位

A：那个 widget 类型在 P0 清洗时被删除（旧业务复合控件）。
- 如果是 `manual-cylinder-block` 等：用 Faceplate 体系的「我的 Faceplate → [内置] 气缸」拖一个新实例取代
- 如果是 `motor` / `alarm-banner` 等：用 I/O 域 + Appearance 动画自行组合实现

### Q4：模拟模式下趋势曲线不动

A：检查趋势 trace 配置的 variable 字段是否填了 `SimTag1` 这类模拟变量名。模拟模式只推 SimTag* / SimBool* / SimText* 这些约定 tag。

### Q5：自动程序生成后 Git 没 push

A：检查 Git 拉取面板「**生成程序后自动 commit + push 到远端**」是否勾选。未勾选时只在本地工作区追加文件，不会自动提交。

### Q6：Faceplate 实例显示「未知 Faceplate」

A：工程文件可能引用了被删除的 Faceplate ID。打开 Faceplate 编辑模式，检查左栏 Faceplate 列表是否含该 ID。如果是内置但被误删，重启软件会自动重新注入。

## 五、Git 代码管理（完整版）

软件深度集成 Git，用于把生成的 PLC `*.st` 源代码与工程仓库同步。流程覆盖：**克隆 / 拉取 / 自动追加 / Diff / 自动 commit / 自动 push** 全链路。

### 5.1 设计哲学

- **不破坏现场代码**：生成代码默认**追加**到现有 `.st` 文件而非覆盖（按映射规则定位段落）
- **不强制本地清理**：默认保留本地未提交改动，避免现场工程师手改丢失
- **不接管 PLC 工程文件**：默认**不拉取** `.project` 文件（PLC IDE 可能锁定）
- **可对比**：生成前后可弹窗 side-by-side 看差异

### 5.2 字段详解

位置：「手动程序生成」左栏「**Git 代码拉取**」Expander。

#### 5.2.1 基本配置

| 字段 | 必填 | 说明 / 示例 |
|---|---|---|
| **仓库地址 (Repository URL)** | ✅ | `https://github.com/your/repo.git` / `git@gitee.com:org/proj.git` / 局域网 `http://gitlab.local:8090/op/code.git` |
| **分支 (Branch)** | ⭕ 可选 | 留空使用远端默认分支（master/main/develop 都行） |
| **保存目录** | ✅ | 本地路径。**首次拉取时若目录非空且非 Git 仓库将报错**——务必给个全新空目录或已有 Git 仓库的目录 |
| **项目文件夹 / 分支名** | ⭕ 可选 | 进阶用法：填如 `OP80` 后软件会克隆到 `<保存目录>/OP80/`，**并自动 checkout 到同名本地分支 OP80**（不存在则基于源分支创建）。多工位独立分支管理利器 |
| **代理地址 (HTTP Proxy)** | ⭕ | 场内代理时填，如 `http://proxy.local:8080`。底层通过 `HTTP_PROXY` / `HTTPS_PROXY` 环境变量传给 git 子进程 |

#### 5.2.2 认证（私有仓库展开）

| 字段 | 说明 |
|---|---|
| **用户名** | Git 用户名（GitHub: 邮箱或用户名 / Gitee: 用户名） |
| **访问令牌 / Personal Access Token** | **不要填密码**，填 PAT（GitHub Settings → Developer settings → Personal access tokens；Gitee 个人设置 → 私人令牌）。Token 至少需要 `repo` 权限 |

⚠️ 安全提示：Token 写入本地 `config/git-pull.json`（明文），请妥善保管本地配置文件 + 设置 Token **过期时间**（如 90 天）+ 用最小权限。

#### 5.2.3 工作流复选框

| 选项 | 默认 | 详细说明 |
|---|---|---|
| **生成程序后自动追加到 Git 目录对应 `.st` 文件** | ✅ 开 | 按映射规则把生成的 PLC 代码段落写入 Git 目录的对应文件。映射规则见 §5.4 |
| **拉取 `.project` 工程文件** | ❌ 关 | 默认 `.gitignore` 行为：拉取时 `*.project` 不会检出到本地，避免覆盖现场 PLC IDE 锁定的工程文件 |
| **强制覆盖本地 (`reset --hard origin/branch` + `clean -fd`)** | ❌ 关 | ⚠️ **危险**：勾选后拉取时丢弃本地所有未提交改动（含生成追加内容、现场临时改动），把工作区重置成远端分支的状态。仅在确认本地无价值改动时用 |
| **拉取后把新分支推送到远端 (`git push -u origin branch`)** | ❌ 关 | 如果配了「项目文件夹 / 分支名」自动创建了本地新分支，勾选后会自动 `push -u` 上传到远端。需要仓库写权限 |
| **生成程序后自动 commit + push 到远端** | ❌ 关 | 每次「生成程序」结束如果产生了追加内容，自动 `git add -A → commit → push`。无变更时不产生空 commit |
| **自动 commit message 模板** | `HMI 生成程序 ({op}) @ {time}` | 模板变量：`{op}` 替换为当前工位号、`{time}` 替换为提交时间戳。仅在上面那项勾选时显示 |

### 5.3 操作按钮

#### 5.3.1 拉取代码

- 首次（本地目录为空 / 不是 Git 仓库）→ 执行 `git clone <url> <local>` + checkout 分支
- 后续（已是 Git 仓库）→ 执行 `git fetch + git checkout + git pull`
- 配了「项目文件夹 / 分支名」→ 自动 checkout / 创建分支
- 勾「强制覆盖本地」→ 额外执行 `reset --hard` + `clean -fd`
- 勾「拉取后推送新分支」→ 额外 `push -u` 新分支到远端
- 全过程**实时日志**输出到下方「执行日志」面板

按钮上方有「**拉取中...**」状态指示。

#### 5.3.2 打开目录

打开本地仓库目录（用系统资源管理器）。配过项目子文件夹的话打开的是子文件夹。

### 5.4 自动追加映射规则（核心）

「生成程序后自动追加到 Git 目录对应 `.st` 文件」勾选后，每次生成的代码按下面映射写入：

| 生成段落 | Git 目录内目标文件 |
|---|---|
| `DBXX_IO`（DB 块声明） | `2.Main/Main_IO.ST` |
| `DI_ACT_Comment` / `DO_ACT_Comment`（注释段） | `<OPxx>/DI_Mirror/ACT_Comment.st` / `<OPxx>/DO_Mirror/ACT_Comment.st` |
| `ACT_*`（动作块） | `<OPxx>/DriveControl/ACT_*.st` |
| `OPxx_Graph_ACT_AutoRunST<N>` | `<OPxx>/Graph/ACT_AutoRunST<N>.st`（自动程序） |
| `OPxx_Graph_ACT_InitST<N>` | `<OPxx>/Graph/ACT_InitST<N>.st`（初始化程序） |

> 实际映射可能因 PLC IDE 工程结构而异，软件会智能识别现场目录结构。如果映射不命中，写入失败会在「执行日志」提示。

### 5.5 Diff 视图（D2）

右栏「程序预览」面板中：
- 选择刚生成的某个 `.st` 文件
- 点击「**对比 Git**」按钮
- 弹出 side-by-side 对比窗：**左侧 Git 当前版本 / 右侧 本次生成**，**红色** = 删除行 / **绿色** = 新增行 / **黄色** = 改动行
- 滚动同步、行号对齐

适合上传前最后审查。

### 5.6 典型工作流场景

#### 场景 A：第一次部署（空仓库 → 完整推送）

1. 在 GitHub/Gitee 建新仓库，拷贝 URL
2. 软件填仓库地址 + 本地全新空目录 + 留空分支（用默认）
3. 私有仓库：展开认证填用户名 + PAT
4. 勾「生成程序后自动追加」+「自动 commit + push」
5. 点「拉取代码」（首次会 clone）
6. 跑「生成程序」→ 代码自动写入对应路径 → 自动 commit → push
7. 在远端仓库 Web 端查看 commit

#### 场景 B：现场维护（增量同步）

1. 之前已配置好仓库，软件启动后已读 `config/git-pull.json` 配置
2. 点「拉取代码」拿最新版本到本地
3. 改 IO 表或 SFC 步骤
4. 跑「生成程序」→ 用「对比 Git」审查差异 → 确认无误
5. 若勾了自动 push，直接到远端；否则手动用 git 工具 commit

#### 场景 C：多工位独立分支（产线多工位并行）

1. 工位 OP10 工程师配「项目文件夹 / 分支名」=`OP10`
2. 工位 OP20 工程师同仓库配「项目文件夹 / 分支名」=`OP20`
3. 两人各自拉取 → 自动各自创建本地分支 → 互不干扰
4. 主仓库通过 `git merge` 合并各分支到 main

#### 场景 D：场内无外网（局域网 GitLab + 代理）

1. 仓库地址用局域网 GitLab：`http://gitlab.local:8090/op/code.git`
2. 如经过代理：填「代理地址」`http://proxy.factory.local:8080`
3. 其余正常操作

### 5.7 错误诊断

「执行日志」面板会输出每条 git 子命令的原文 + 退出码 + stderr。常见错误：

| 错误 | 原因 | 解决 |
|---|---|---|
| `fatal: destination path 'X' already exists and is not an empty directory` | 首次克隆但本地目录非空 | 清空目录或选其他目录 |
| `fatal: Authentication failed` | 用户名 / Token 错 | 检查认证字段；token 是否过期；GitHub 的 password 已弃用必须用 PAT |
| `Could not resolve host` | 网络 / DNS 问题 | 配代理 / 检查仓库 URL |
| `error: Your local changes to the following files would be overwritten by merge` | 本地有未提交改动且与远端冲突 | 勾「强制覆盖本地」（丢弃改动）或手动 git stash |
| `! [rejected] master -> master (fetch first)` | 远端有新 commit，本地落后 | 先拉取再 push |
| `Permission denied (publickey)` | 用了 SSH URL 但没配 SSH key | 改用 HTTPS URL + PAT |

### 5.8 安全与最佳实践

✅ **推荐做法**：
- Token 用最小权限（仅 `repo` 一项）+ 设过期时间 90 天
- 重要工程用受保护分支（protected branch）禁止强推
- 现场机器关机前点「打开目录」检查日志，确认所有 commit 已 push
- 定期用「对比 Git」审查生成代码避免覆盖手改

❌ **避免**：
- ❌ 把 GitHub 主账号密码填进认证（已被 GitHub 弃用且不安全）
- ❌ 长期勾「强制覆盖本地」（一不小心丢现场临时改动）
- ❌ 拉取 `.project` 文件（除非确认 PLC IDE 已关闭）
- ❌ 在生产环境直接对 master 分支自动 push（用 feature 分支 + PR review）

### 5.9 配置持久化

所有 Git 配置（含 Token）保存在 `config/git-pull.json`：

```json
{
  "GitRepositoryUrl": "https://github.com/your/repo.git",
  "GitBranch": "",
  "GitTargetFolder": "C:\\PLC\\projects\\factoryA",
  "GitProjectFolderName": "OP70",
  "GitUsername": "engineer@example.com",
  "GitAccessToken": "ghp_xxxxxxxxxxxxxxxxxxxx",
  "GitProxyUrl": "",
  "IsSyncGeneratedToGitEnabled": true,
  "IsIncludeProjectFilesOnPullEnabled": false,
  "IsForceResetLocalEnabled": false,
  "IsPushProjectBranchToRemoteEnabled": false,
  "IsCommitAndPushAfterGenerateEnabled": true,
  "GitAutoCommitMessageTemplate": "HMI 生成程序 ({op}) @ {time}"
}
```

下次启动自动加载。**部署到新机器只需要拷贝这个文件 + 仓库配置就生效**。
