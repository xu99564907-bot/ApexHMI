# PLC OPC UA HMI Designer - 软件需求文档

## 1. 概述

### 1.1 软件名称

PLC OPC UA HMI Designer

### 1.2 软件定位

一款基于 WPF 和 OPC UA 协议的工业人机界面(HMI)设计与监控软件，支持可视化界面设计、PLC 数据通讯、IO 表导入及程序自动生成。采用成熟的 Prism MVVM 框架，支持多平台 PLC 配置。

### 1.3 运行环境

- **操作系统**: Windows 10/11 (64位)
- **运行框架**: .NET Framework 4.8
- **开发技术**: C# / WPF / OPC UA / Prism

## 3. 核心功能需求

### 3.1 界面布局系统

#### 3.1.1 整体布局架构


| 区域   | 位置      | 功能描述                            |
| ---- | ------- | ------------------------------- |
| 标题栏  | 顶部      | 自定义无边框窗口，包含菜单栏和窗口控制按钮           |
| 导航区  | 顶部      | 9个主要功能模块的图标导航按钮                 |

#### 3.1.2 导航模块

软件包含以下 9 个主要功能模块：

1. **主界面** - 系统概览和快捷入口
2. **监控** - 实时数据监控(IO、变量、流程)
3. **手动操作** - 设备手动控制(气缸、电机、轴等)
4. **参数设定** - 系统参数和工艺参数配置
5. **配方管理** - 产品配方创建与管理
6. **报警画面** - 实时报警和历史报警查询
7. **登录** - 用户权限管理
8. **设计器** - 可视化界面设计和程序生成

#### 3.1.3 界面特性

- 支持运行态/设计态切换
- 响应式布局，自适应不同分辨率
- 支持窗口最大化/还原/最小化
- 无边框窗口设计，自定义标题栏
- 使用 Prism RegionManager 实现视图导航

### 3.2 多平台 PLC 通讯服务

#### 3.2.1 PLC 接口设计

采用接口隔离设计，支持多种 PLC 品牌：

```csharp
public interface IPlc : IDisposable
{
    Guid Id { get; }
    int Index { get; set; }
    CommunicationType CommunicationType { get; }
    string EndpointUrl { get; }
    string Name { get; }
    bool IsConnected { get; }
    
    event Action<object, bool> ConnectChangedEvent;
    
    void Connect();
    Task ConnectAsync();
    void Disconnect();
    
    // 数据读写
    object ReadNode(string nodeId);
    Task<object> ReadNodeAsync(string nodeId);
    T ReadNode<T>(string nodeId);
    Task<T> ReadNodeAsync<T>(string nodeId);
    Dictionary<string, object> ReadNodes(string[] nodeIds);
    Task<Dictionary<string, object>> ReadNodesAsync(string[] nodeIds);
    
    bool WriteNode<T>(string nodeId, T value);
    Task<bool> WriteNodeAsync<T>(string nodeId, T value);
    bool WriteNodes(Dictionary<string, object> nodeValues);
    Task<bool> WriteNodesAsync(Dictionary<string, object> nodeValues);
    
    // 订阅
    void SubscribeNodes(string key, List<string> nodeIds, Action<(string key, string nodeId, object value)> dataChangeHandler);
}
```

**PLC 配置信息：**

```csharp
public class PlcInfo
{
    public Guid Id { get; set; }
```

### 3.3 IO 表导入与程序生成

#### 3.3.1 IO 表导入功能

**支持文件格式：**

- CSV 文件(.csv)
- 文本文件(.txt)
- Excel 文件(.xlsx)

**支持编码格式：**

- UTF-8
- Unicode
- GB2312/GBK (中文)

**数据格式支持：**

1. **结构化格式** (推荐)
  - 输入模块、输入地址、输入工位、输入变量注释、输入备注
  - 输出模块、输出地址、输出工位、输出变量注释、输出备注
2. **模块表格式**
  - 模块编号、输入地址、工位、变量注释、备注
  - 输出地址、工位、变量注释、备注
3. **传统格式**
  - 输入地址、输入变量、输出地址、输出变量

**导入特性：**

- 自动检测文件编码
- 自动识别表头格式
- IO 地址自动规范化(IX/QX 格式)
- 自动填充缺失的 IO 位

#### 3.3.2 IO 表管理


| 功能项  | 需求描述               |
| ---- | ------------------ |
| 数据预览 | 表格形式预览导入的 IO 数据    |
| 数据编辑 | 支持在界面中编辑 IO 表数据    |
| 保存导出 | 支持保存为 CSV/Excel 格式 |
| 自动保存 | 支持定时自动保存功能         |


#### 3.3.3 程序生成功能

**生成配置：**


| 配置项      | 说明                |
| -------- | ----------------- |
| PLC 模板   | 汇川中型PLC / 汇川小型PLC |
| 工位号      | OP 编号，如 OP10      |
| 控制 DB 乘数 | DB 编号计算参数         |
| 控制 DB 偏移 | DB 编号偏移量          |
| 驱动 DB 偏移 | 驱动 DB 偏移量         |


**生成程序文件：**


| 文件名                | 内容描述        |
| ------------------ | ----------- |
| DBxx_IO.txt        | IO 变量声明程序   |
| DI_ACT_Comment.txt | 输入注释监控程序    |
| DO_ACT_Comment.txt | 输出注释监控程序    |
| ACT_Cylinder.txt   | 气缸控制程序      |
| ACT_Vacuum.txt     | 真空控制程序      |
| ACT_Sensor.txt     | 传感器程序       |
| ACT_Motor.txt      | 电机控制程序      |
| ACT_Axis.txt       | 轴控制程序       |
| ACT_Rotdisk.txt    | 转盘控制程序      |
| ACT_Epson.txt      | Epson 机器人程序 |
| ACT_Kuka.txt       | KUKA 机器人程序  |


**对象识别规则：**

- 气缸(Cylinder): CY + 数字 / 包含"气缸"关键字
- 真空(Vacuum): VAC + 数字 / 包含"真空"关键字
- 传感器(Sensor): SENSOR + 数字 / 包含"传感器"关键字
- 电机(Motor): MOTOR + 数字 / 包含"电机"关键字
- 轴(Axis): AXIS + 数字 / 包含"轴/伺服"关键字
- 转盘(Rotdisk): ROTDISK + 数字 / 包含"转盘"关键字
- Epson 机器人: EPSONROB + 数字 / 包含"EPSON"关键字
- KUKA 机器人: KUKAROB + 数字 / 包含"KUKA"关键字

- 支持 OPC UA 连接断线重连
- 数据自动保存机制
- 异常错误提示和日志记录

### 4.3 易用性需求

- 直观的图形化界面
- 拖拽式操作
- 实时预览效果
- 中文界面支持

### 4.4 扩展性需求

- 支持自定义 PLC 模板
- 支持扩展新的控件类型
- 支持自定义命名规则
